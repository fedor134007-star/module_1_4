Практика:

Есть два типа потоков: водородные (H) и кислородные (O). Необходимо синхронизировать их работу так, чтобы они проходили через "барьер" и образовывали молекулы воды строго по правилу: на одну молекулу - 2 H и 1 O. Потоки должны группироваться строго тройками (H, H, O) и сразу "связываться" (bond) в рамках одной тройки, прежде чем начнет связываться следующая.
Что требуется обеспечить
Потоки пропускаются через барьер только полными тройками: в каждой тройке - ровно 2 водорода и 1 кислород.
Никакое связывание следующей тройки не начинается, пока вся текущая тройка не прошла барьер и не "связалась".
Если пришел кислород, но нет двух водородов - он ждет.
Если пришел водород и он "лишний" (нет свободного кислорода или уже есть 2 H, ожидающих O) - он ждет.





Вам дан класс:
class H2O {

    public H2O() { }

    public void hydrogen(Runnable releaseHydrogen) throws InterruptedException {
        // releaseHydrogen.run() печатает "H"
        releaseHydrogen.run();
    }

    public void oxygen(Runnable releaseOxygen) throws InterruptedException {
        // releaseOxygen.run() печатает "O"
        releaseOxygen.run();
    }
}


Метод hydrogen вызывается водородным потоком и должен выпустить символ "H" ровно тогда, когда этот поток входит в состав корректной тройки.
Метод oxygen вызывается кислородным потоком и должен выпустить символ "O" ровно тогда, когда этот поток входит в состав корректной тройки.
Ваша задача - добавить синхронизацию (sem/lock/condvar/barrier и т.п.), чтобы вывод всегда можно было разбить на тройки, каждая из которых содержит ровно два H и один O, без перемешивания тройками.


Примеры
Вход: water = "HOH"
 Возможный вывод: "HHO" (также корректны "HOH", "OHH").
Вход: water = "OOHHHH"
 Возможный вывод: "HHOHHO"
 (Также корректны: "HOHHHO", "OHHHHO", "HHOHOH", "HOHHOH", "OHHHOH", "HHOOHH", "HOHOHH", "OHHOHH" и др., если каждые 3 символа образуют валидную молекулу).


Важно: проверяется не конкретный порядок символов, а то, что каждая подряд идущая тройка символов в итоговой строке - это перестановка ["H","H","O"].
Ограничения
3 * n == water.length
1 <= n <= 20
water[i] ∈ {'H','O'}
Ровно 2 * n символов 'H' и ровно n символов 'O'.


Суть задачи (в терминах синхронизации)
Нужно реализовать координацию группового пропуска: только когда набраны 2 H + 1 O, разрешить этой тройке напечатать "HHO" в любом порядке, но целиком, не смешивая с другой тройкой.


После завершения "связывания" текущей тройки - счетчики обнуляются, и можно пропускать следующую тройку.


Подсказки к подходам (без решения)
Часто используют семафоры/счетчики для ограничения числа одновременно проходящих H и O.


Удобен механизм барьера на 3 участника или ручное "барьерное" согласование на условиях/мониторах.


Нужны два уровня контроля:


Состав тройки (не более 2 H и 1 O до "сброса"),


Пакетное завершение (все трое выпускаются и только затем разрешается формировать следующую тройку).


Этого описания достаточно, чтобы приступить к реализации синхронизации, не меняя сигнатуры методов и не трогая логику печати, кроме обертки синхронизации вокруг releaseHydrogen.run() и releaseOxygen.run().
